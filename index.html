Security Research

Prompt injection, agent abuse,
and how we defend.

Frontier models and agentic systems are exposed to prompt injection and tool abuse. We apply OWASP LLM01–aligned test cases and defenses in evaluation and red-teaming so your deployments stay secure.

01

Prompt injection

Attackers inject instructions via user input or retrieved content to override system behavior, leak system prompts, or bypass safety. We test direct and indirect injection, role override, and delimiter confusion.

02

Agent & tool abuse

LLM agents that call tools can be manipulated into misusing APIs—wrong recipients, privilege escalation, or destructive actions. We evaluate tool-call validation and least-privilege design.

03

Mitigations

Input/output validation, sandboxing, monitoring, and structured red-teaming. Our writeup documents test cases and defenses we use in engagements—and how to harden your stack.
